# <font style="color:rgb(38, 38, 38);">本周学习的知识分享</font>

* #### DSA(Digital Signature Algorithm)

  DSA，即数字签名算法，和RSA一样，DSA也是一种非对称加密，但与rsa不同的是，dsa没有预设一套加密-解密机制，也就是说只可能单向验证哈希的正确性而没法用来传递信息

  > 常见例题如‘k重复使用私钥泄露‘
  > 也就是在两次加密用了同一个本该不同的随机数k导致的私钥泄露
  >
  > 全局公钥参数p,q和g(暂时不用管，总之是已知的)
  >
  > 已知签名(r, s1)和(r, s2)和两次的哈希h1,h2
  > S1 = k^(-1) * (h1 + x * r) mod q
  > S2 = k^(-1) * (h2 + x * r) mod q
  >
  > 也就是说有s1 - s2 = k^(-1) * (h1 + x * r - h2 - x * r) mod q
  > 也即s1 - s2 = k^(-1) * (h1 - h2) mod q
  > 得到k = (h1 - h2) * (s1 - s2)^(-1) mod q
  >
  > 别忘了h1h2s1s2和q都是已知的
  > 也就是说此时k已经能被计算了，算出k后
  > 此时s1或s2就只剩下x一个未知数了
  > 再把k反代回去再移项什么的最后能求出x
  > 好了，私钥x就得到了
